## 各目录职责详解

### `public/`
所有放在这里的文件会被**原样复制**到构建输出目录（`dist/`），不会被 Vite 处理。

### `src/components/ui/`
shadcn/ui 组件放在这里。这些文件是你项目的源码（不是 node_modules 里的），可以随意修改。这是 shadcn/ui 设计哲学的核心——"你拥有这些代码"。

### `src/pages/`
每个文件代表一个页面。页面 = 用户看到的一个完整屏幕。通过 React Router 配置路由映射。

### `src/layouts/`
布局组件，定义页面的"外壳"——导航栏、侧边栏、页脚等。使用 React Router 的 `<Outlet />` 作为子页面内容的占位符。

### `src/hooks/`
自定义 Hook = 可复用的逻辑片段。比如"从 API 获取数据"这个逻辑，多个页面都可能用到，抽成一个 `useFetch.ts` Hook。

### `src/lib/`
"lib" = library 的缩写。放工具函数和 API 客户端的封装。

### `functions/`
Cloudflare Pages Functions 的目录。每个 `.ts` 文件导出 `onRequest` 函数，自动成为一个 HTTP 端点。后期加入。

### `wrangler.toml`
Wrangler 是 Cloudflare 的 CLI 工具。后期创建。
