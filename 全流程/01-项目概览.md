## 核心原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | 官方文档优先 | 一切操作以官方文档为第一参考，不用二手教程、不凭记忆 |
| 2 | 前端先行 | 先开发前端，后端和数据库后续再建，不提前引入复杂度 |
| 3 | 边缘函数 | 使用 Cloudflare Pages Functions（`/functions` 目录），与前端同项目、同部署 |
| 4 | 设计优先 | 使用成熟的 UI 库（shadcn/ui），不做重复造轮子 |
| 5 | 纯 React | 不需要 Next.js，React + Vite 即可满足需求 |

---

## 技术选型

| 类别 | 选择 | 原因 |
|------|------|------|
| 前端框架 | React 19 | 社区标准，生态最丰富 |
| 构建工具 | Vite | 官方推荐，开发时热更新快，生产打包基于 Rollup |
| 语言 | TypeScript | 类型安全，减少运行时错误 |
| UI 组件库 | shadcn/ui | 基于 Radix UI + Tailwind，高质量、可定制、不引入额外 npm 包 |
| 样式 | TailwindCSS v4 | shadcn/ui 依赖，原子化 CSS，不用手写样式文件 |
| 动画库 | animejs v4 | 用户指定，轻量级 JS 动画引擎 |
| 边缘函数 | Cloudflare Pages Functions | 与 Pages 部署天然集成，`/functions` 目录即 API |
| 部署 | Cloudflare Pages | 全球边缘网络，免费额度足够个人项目 |
| 通信 | fetch / REST API | 简单直接，前端和后端之间标准通信方式 |

---

## 暂不引入的技术

| 技术 | 原因 |
|------|------|
| SSE | 用户决定不用。Cloudflare Workers 对 SSE 支持有限制（CPU 时间、连接时长、已知 Bug），等有明确场景再评估 |
| Next.js | 不需要。Vite + React 已经覆盖需求，Next.js 在此项目中是过度引入 |
| Prisma / 数据库 | 后续再建，不阻塞前端开发 |
| 状态管理库（Zustand/Redux） | 先用 React 内置的 useState + useContext，不够再加 |

## 架构概览

```
浏览器（用户）
    │
    │  访问 https://你的域名
    ▼
┌───────────────────────────────────────┐
│         Cloudflare Pages              │
│                                       │
│  静态资源（Vite 打包的 HTML/JS/CSS）    │
│  ┌─────────────────────────────────┐  │
│  │ React SPA                       │  │
│  │ ├── 页面路由（React Router）     │  │
│  │ ├── shadcn/ui 组件库            │  │
│  │ └── animejs 动画                 │  │
│  └─────────────────────────────────┘  │
│                                       │
│  边缘函数（/functions 目录）           │
│  ┌─────────────────────────────────┐  │
│  │ /functions/api/                 │  │
│  │ ├── 认证、鉴权                   │  │
│  │ ├── 数据读写（后续连数据库）      │  │
│  │ └── 其他服务端逻辑               │  │
│  └─────────────────────────────────┘  │
│                                       │
└───────────────────────────────────────┘
```

- **静态资源**：Vite 构建后的纯文件，由 Cloudflare 全球 CDN 分发，访问速度快
- **边缘函数**：`/functions` 目录下的每个文件自动变成一个 API 端点，运行在 Cloudflare 的边缘网络

## 开发流程

```
本地开发：
  阶段1（当前）: 只跑 vite dev → localhost:5173
      纯前端开发：页面、组件、样式、动画
      用 mock 数据模拟后端行为

  阶段2（后续）: vite dev + wrangler dev 同时跑
      加入 /functions 边缘函数
      Vite 代理转发 API 请求到 Workers

生产部署：
  git push → Cloudflare Pages 自动构建和部署
  或手动: wrangler pages deploy
```
