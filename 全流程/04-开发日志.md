## 2026-07-20

### Step 1：技术选型和架构讨论
- **做了什么**：讨论边缘函数方案，对比 Next.js vs 纯 React + Vite，评估 SSE 在 Cloudflare Workers 上的可行性
- **结论**：
  - 框架：React + Vite（不用 Next.js）
  - UI：shadcn/ui + TailwindCSS v4
  - 动画：animejs v4
  - 边缘函数：Cloudflare Pages Functions（后期加入）
  - 通信：REST API（fetch），不用 SSE
  - 开发策略：先纯前端，后端/数据库后续再建
- **原因**：
  - Vite 更轻量，项目不需要 Next.js 的 SSR/ISR 能力
  - Cloudflare Workers 对 SSE 支持有限制（CPU 时间、连接时长、已知 Bug），风险大于收益
  - shadcn/ui 提供高质量预制组件，减少手写 UI 的时间

### Step 7：animejs v4 教学 Demo
- **做了什么**：创建 `src/pages/AnimeDemo.tsx`，包含 4 个教学 Demo：
  - **Demo 1：基础属性动画** — translateX + rotate + scale，演示 `animate(targets, params)` 核心 API
  - **Demo 2：Stagger（错开动画）** — 5 个元素逐个出现，使用 CSS 选择器 + `stagger` 参数
  - **Demo 3：缓动函数对比** — linear / easeOut / easeIn / easeInOut 四种节奏对比
  - **Demo 4：动画控制** — 暂停/恢复/反转/停止，演示 `JSAnimation` 对象的方法
- **教学风格**：每个 Demo 都有"查看代码"折叠区，展示实际代码 + 注释讲解
- **技术细节**：
  - animejs v4 API 变化：`animate(targets, params)` 替代 v3 的 `anime({ targets, ... })`
  - `stagger` 参数让多个元素按间隔排队执行
  - `JSAnimation` 对象提供 `.pause()` `.resume()` `.reversed` `.cancel()` `.complete()` 方法
  - 使用 `useRef` 获取 DOM 元素（不触发重新渲染），`useState` 管理 UI 状态
- **结果**：✅ TypeScript 编译通过，所有 Demo 可交互
